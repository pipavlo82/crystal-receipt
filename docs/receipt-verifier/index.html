Receipt Verifier

RECOMPUTE, NOT TRUST
Independently recomputable evidence for ReceiptOS-shaped receipts — every step below runs in your browser, not on a server.
Input
Verification ladder
Paste a receipt or load a sample, then run verification.
What this checks: canonicalization + recompute of receipt_root, Merkle-path closure to the declared merkle_root, and presence of PQ signer metadata. Canonicalization here is a reference implementation (sorted-key JSON → UTF-8 → SHA-256) for demonstration — it is not guaranteed byte-identical to the authoritative crystal-receipt canonicalizer. A receipt only earns PROOF class when root and Merkle path both verify; sig_pq is carried as signer/provenance metadata and does not by itself upgrade the class.
